A from-scratch implementation of recursive backtracking, the algorithm that searches a problem space by trying a choice, recursing, and undoing it the instant it breaks a rule. This project applies it to Sudoku — with real-time hash-set validation and a fully interactive 9×9 board.

02 — Under the Hood

Two structures doing the heavy lifting

The solver leans on two core DSA ideas: depth-first recursive backtracking to search the solution space, and hash sets to validate constraints in constant time.

01

Recursive Backtracking

The solver scans for the next empty cell, tries each candidate digit 1–9, and recurses into the next empty cell. If a branch leads to a dead end, it backtracks: undo the last placement and try the next candidate. This depth-first search guarantees a correct solution (when one exists) without brute-forcing every permutation. 

function solve(grid):
    cell = findEmptyCell(grid)
    if cell is null:
        return true            // solved
    for digit in 1..9:
        if isValid(grid, cell, digit):
            place(grid, cell, digit)
            if solve(grid):
                return true
            undo(grid, cell)    // backtrack
    return false                // trigger backtrack upstream
02

Hash Set Validation

Every row, column, and 3×3 box keeps its own Set of digits already placed. Checking whether a digit is safe to place is an O(1) lookup against three sets instead of re-scanning 27 cells on every attempt, which is what keeps the search responsive even as it backtracks heavily. 

const rows = Array.from({length:9}, () => new Set());
const cols = Array.from({length:9}, () => new Set());
const boxes = Array.from({length:9}, () => new Set());

function isValid(r, c, val) {
  const b = boxIndex(r, c);
  return !rows[r].has(val)
      && !cols[c].has(val)
      && !boxes[b].has(val);
}

03 — Algorithmic Cost

What backtracking actually saves you

Approach Time Complexity Space Complexity Why
Brute force (try every digit, every cell) O(9^81) O(1) No pruning — every dead branch is explored to completion.
Recursive backtracking (this project) O(9^m) worst case* O(m) recursion depth Prunes a branch the moment a constraint is violated, where m is the count of empty cells.
Constraint validation per placement O(1) O(1) per set Hash set lookups replace O(n) row/column/box scans.

*Worst case remains exponential in theory, but constraint propagation from filled cells prunes the practical search tree dramatically for valid puzzles.
